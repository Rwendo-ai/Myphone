## 3. Architecture state

### Subscription model (v3 — shipped)
- 5 monthly tiers: Text $4 · Voice $5 · Lipsync Low $10 · Lipsync High
  $25 · Ultra $29.99
- 2 yearly variants
- 1 lifetime ($399)
- 3 token packs (60 / 200 / 800 AI credits; lifetime buyers get +15%)
- Live tier derivation from RC entitlements in `lib/entitlements.ts`
- `DEV_UNLOCK_ALL` flag still on for dev builds; needs stripping before
  prod

### Owner access list
5 UUIDs in `OWNER_USER_IDS` bypass every gate:
- `60f2f862-a58e-4afc-ae1d-d2bcd39ae5b4` — bowencooney@gmail.com (you)
- `47f6148a-0131-48d3-beac-f56ea7f08727` — aussi3bs@gmail.com (you)
- `d3737a86-...` — charmiephotos1@gmail.com (girlfriend)
- `bbf622c9-...` — johnsonmakoti@hotmail.co.uk (tester)
- `4b868d72-...` — jmakoti35@gmail.com (tester)

### UI architecture
- Floating profile-left + cart-right header on all 5 main tabs (Home /
  AI Chat / Companions / Travel / Learn)
- New `/cart` route surfaces the full v3 pricing model
- Connections feed is open (no destination filter) with Follow/Unfollow
  on every post card

### RevenueCat client
- SDK initialised in `lib/purchases.ts`
- `isTestKey()` guard skips `Purchases.configure()` when key has `test_`
  prefix — prevents the production-key dialog from killing the app
- Wires entitlements through `useEntitlements()` hook
- Pending: webhook Edge Function + DB-mirror fallback

### Streaming content architecture
- `course-content` bucket: lessons + flipcards + reinforcement cards
- `travel-content` bucket: phrasebooks + cultural guides + safari decks
- All content lazy-loaded JSON over Supabase Storage
- Per-speaker variants picked at load time; fallback to canonical when
  missing
- Device cache at `documentDirectory/<bucket>/...` with in-memory
  shortcut

### XP economy
- Award flow: `award_xp(p_user_id, p_event_type, p_event_id, p_amount)`
  RPC; idempotent per `(user, event_id)`
- Per-lesson award: 25 XP for non-final lesson, 50 XP for last lesson of
  module
- Daily-XP display bug: Learn tab shows `xp % goal` (wrong)

### Supabase schema state
Tables in place:
- `auth.users` (managed) · `user_profiles` · `companions` · `xp_events`
- `connections_posts` · `connections_follows`
- `lesson_translation_cache` (intended for lessons; works for cards too)

Pending:
- `user_entitlements` — DB mirror of RC entitlement state
- `user_credits` — AI credit balance
- `user_quota` — free-tier daily Rwen msgs + per-lesson AI msgs

### Edge Functions
- `award-xp` ✓ deployed
- `lesson-tutor` ✓ deployed (Anthropic-backed)
- `revenuecat-webhook` ⏳ skeleton at
  `supabase/functions/revenuecat-webhook/`; not yet deployed
- `translate-cards` ⏳ designed in
  `docs/REINFORCEMENT-CARDS-TRANSLATION.md`; skeleton not yet written
