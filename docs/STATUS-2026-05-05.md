## TL;DR — what changed today

We pivoted lesson delivery from "bundled in the JS binary" to "streamed from
Supabase Storage on demand, cached locally". This is a load-bearing change:

- The 576 authored lessons across 13 courses now live in the `course-content`
  Supabase Storage bucket at `lessons/<courseId>/<speakerId>/<lessonId>.json`.
- The repo no longer contains lesson bodies — only metadata manifests.
- The lesson screen downloads on first open, caches per-device, and reads
  from cache thereafter. No auto-eviction; explicit uninstall only.
- All wired and verified working end-to-end on device for Shona, French,
  Tagalog, and Knowing Yourself courses.

The change is **uncommitted**. Recovery git tag `v0.2-pre-streaming-refactor`
exists at the pre-pivot commit if anything goes sideways.

## What we discussed and decided this session

### Streaming architecture (locked)

Trade-off accepted: brief per-lesson load on first open, in exchange for a
30-50 MB lighter binary and the ability to ship/edit lessons without an EAS
rebuild. Lazy translation (next phase) writes to the same path scheme so
non-English speakers eventually get a cache hit too.

Cache eviction policy: **explicit uninstall only**. No LRU, no cache-pressure
eviction. Surprising users with disappeared lessons mid-flight is worse than
the disk-space cost.

### What stays as-is (not pivoting)

- **The 7-phase Rwendo Method** (hook → chunks → pattern → practice →
  dialogue → recall → mission) is the lesson format for all 576 authored
  lessons. We are NOT replacing it with AI-conversational lessons.
- The AI surface is alongside — Phase 8 (post-lesson 6-msg Claude chat) and
  the AI Companion tab (separate, fully conversational).

   [:182](../lib/lesson-loader.ts#L182).

2. White screen on unit tap — `getUnit(id)` only knew the legacy hand-authored

   [data/lessons.ts:getCourseUnit](../data/lessons.ts) parses synthesized IDs

3. Wrong unit title ("Greetings & Respect") on Knowing Yourself — Module 1

   meta in [lib/manifests.ts](../lib/manifests.ts) `SELF_DEV_MODULE_META`,
   threaded through `getUnitsForCourse(courseId, lessons, moduleMeta?)`.

4. Cross-category routing bug — selecting Knowing Yourself, switching to
   Languages, tapping a Shona lesson would load `know-yourself/<lesson>`
   because `activeCourseId` was stale. Fix: unit screen passes `courseId`
   via route param; lesson screen reads
   `lessonCourseId = routeCourseId ?? activeCourseId`.
   See [app/lesson/[id].tsx:30-36](../app/lesson/[id].tsx#L30) and
   [app/unit/[id].tsx:43](../app/unit/[id].tsx#L43).

5. Paywall blocking dev preview — `DEV_UNLOCK_ALL` only bypassed
   `canAccessLesson`, not the Learn-tab owned/locked pill split. Fix: `isOwned`

### 3. Lazy translation Edge Function
The big remaining feature. Spec:
- Path: `supabase/functions/translate-lesson/index.ts`
- Auth: requires authenticated user (verify JWT)
- Input: `{ courseId, speakerId, lessonId }`
- Behaviour:
  1. Reject if `speakerId === 'english'` (source language; nothing to do).
  2. Check Storage for `lessons/<courseId>/<speakerId>/<lessonId>.json`. If
     present, return its URL — race-condition guard.
  3. Download `lessons/<courseId>/english/<lessonId>.json` (the source).
  4. Call Claude Haiku 4.5 with a translation prompt that preserves the
     `LessonData` shape exactly (every field translated except IDs). Use
     prompt caching for the schema-preservation instructions.
  5. Validate the response against the `LessonData` shape (basic — required
     fields exist, `chunks` length matches, etc.).
  6. Upload to `lessons/<courseId>/<speakerId>/<lessonId>.json` via service-role.
  7. Return `{ ok: true, path }`.
- Cost target: ~$0.12 / lesson (one-time per language; every subsequent
  same-language user gets a cache hit).
- Client wiring: `lib/lesson-loader.ts` calls this function on the
  english-fallback path BEFORE returning the english variant. If the
  Edge Function succeeds, refetch the speaker variant from Storage and
  return that instead.
- Decision needed: do we eagerly translate when a course is installed, or
  lazily on first open? Recommendation: lazy, to avoid translation cost
  for users who install but abandon a course.

### 7. Phase 8 graceful exit detection
Pending from earlier — when a user says "I need to stop" mid-Phase-8 chat,
detect the intent, save partial progress, award proportional XP, route
back to lesson-completion screen. Heuristic: simple keyword match against

### 8. Three language-pack architecture decisions
Earlier-deferred questions — none blocking, but worth resolving:
- **Kinship asymmetry**: Shona has finer-grained kinship terms than English
  (mukoma vs munin'ina). When Claude lazy-translates an English lesson into
  Shona, we lose those distinctions. Fix: speaker-pack-side glossary that
  the translator prompt consults.
- **Grammar lessons**: Shona's noun-class system (Module 4) is hand-authored.
  When a French speaker opens it, Claude needs to translate explanations
  *of* Shona grammar — different problem from translating dialogue. Decide
  whether to pre-author per non-English speaker or rely on a different
  translation prompt.
- **Cultural framing**: each speaker pack has cultural notes (Shona tsika,
  French formality patterns). Currently English-only on the lesson side;
  decide whether speakers' cultural framing replaces, merges, or layers.

## Open thinking I haven't acted on yet

- **Pre-fetch on course pick** — when user taps a course pill on Learn tab,
  optionally kick off `installCourse` in the background so the unit list
  is instantly responsive. Tension: data-cap-conscious users might prefer
  pull-on-demand. Resolution: gate pre-fetch behind a Profile setting,
  default ON for Wi-Fi, OFF for cellular (need to detect via
  `expo-network`).

- **Translation cache invalidation** — when we tweak a source-English
  lesson, every translated variant in Storage becomes stale. Strategy:
  bump a version field inside the JSON; loader compares cached vs Storage
  version on read; mismatch triggers refetch + re-translate. Defer until
  we actually edit a shipped lesson.

- **Manifest-as-source-of-truth tension** — `manifest.ts` is generated
  from the same TS source files we deleted. Right now the script reads
  from `scripts/<course>-source/` (kept locally, not in git) to regenerate.
  Long-term, source-of-truth should be Storage itself: a script that
  walks the bucket and writes the manifest. Not urgent.
