# Seamless Idle-Video Loops

**Question Bowen asked (2026-05-16):**
> Is there any high-quality consistent option so video loops and face is same?

The current 20 human + 10 animal archetype idle videos were generated by
Kling 1.6 Standard image-to-video on fal.ai. Kling drifts between the
first and last frame even with `return-to-starting-pose` in the motion
prompt — the loop seam shows as a small "jump" or "blackout" between
cycles.

This doc lays out the three honest paths to fix it, cheapest first.
None requires changing the runtime — the player just gets a better
clip in the same Storage path and plays it natively.

---

## Option A — FFmpeg crossfade (post-process existing videos, $0)

Take the existing 10s Kling clip and produce a new file where the
**last N seconds cross-fade back into the first N seconds**. The seam
is baked into the file, hidden inside the crossfade.

```
ffmpeg -i input.mp4 -filter_complex "
  [0:v]split[main][tail];
  [tail]trim=start=9:end=10,setpts=PTS-STARTPTS[ovr];
  [main]trim=start=0:end=9,setpts=PTS-STARTPTS[base];
  [ovr]format=yuva420p,fade=t=out:st=0:d=1:alpha=1[ovrf];
  [base][ovrf]overlay=eof_action=pass[v]
" -map "[v]" -c:v libx264 -crf 20 -an output.mp4
```

Trade-offs:
- ✅ Zero cost. No AI re-generation.
- ✅ Works on every existing video.
- ✅ Loops are guaranteed seamless.
- ⚠️ Loses 1s of usable runtime per clip (10s → 9s).
- ⚠️ The cross-faded second can look very slightly soft / smeared on
  high-motion clips. Fine on idle breathing + blinks.

**Recommendation:** start here. Build `scripts/seamless-loop-idle-videos.ts`
that pulls each idle.mp4 from Storage, runs FFmpeg locally, re-uploads.
~30 seconds per clip locally. Run once. Done.

## Option B — Forward-then-reverse mirror (post-process, $0)

Concatenate the source clip with a reversed copy of itself. The result
is double-length but mathematically guarantees seamless looping (end
of forward = start of reverse = same frame; same at the other join).

```
ffmpeg -i input.mp4 -filter_complex "[0:v]reverse[r];[0:v][r]concat=n=2:v=1[v]" \
  -map "[v]" -c:v libx264 -crf 20 -an output.mp4
```

Trade-offs:
- ✅ Zero cost, zero quality loss in the original frames.
- ✅ Mathematically perfect loop (no fade smudging).
- ⚠️ 2× file size and runtime.
- ⚠️ Blinks/breaths play in reverse on every other cycle — usually
  invisible for these subtle motions but can look weird on clips with
  asymmetric motion.

**Recommendation:** B is more honest than A if you'd rather have crisp
frames than a fade. But you've gotta accept slight reverse weirdness.

## Option C — Regenerate with a different model ($$)

Drop Kling for the idle generation step. Use a model with better loop
consistency:

| Model | Cost / 10s | Notes |
|---|---|---|
| **Hailuo MiniMax i2v-01-live** | ~$0.40 | Specifically tuned for talking-head idles. Loop quality reportedly tighter than Kling on simple motion. |
| **Runway Gen-3 Alpha Turbo** | ~$1.00 | Excellent consistency, more expensive. |
| **Veo 3 (Google)** | ~$1.50 | Top quality, limited access, requires Google waitlist approval. |
| **Stable Video Diffusion 1.1** | ~$0.20 | Self-host capable, lower quality. |

`fal.ai` already hosts MiniMax and Runway — same API, just swap the model
slug in `scripts/generate-archetype-assets.ts`. Roll the dice on a few
archetypes; if they loop noticeably better, swap defaults.

**Recommendation:** try MiniMax for a couple of archetypes (~$0.80 to test
both Sarah and Marcus). Compare side-by-side with the existing Kling
clip. Decide before regenerating all 30.

---

## Decision matrix

| If you want… | Pick |
|---|---|
| Zero cost, fastest, "good enough" loop | **A** (FFmpeg crossfade) |
| Crisp frames, accept slight reverse playback | **B** (FFmpeg mirror) |
| Top quality, willing to spend ~$15 to regenerate all 30 | **C** with MiniMax |
| Top quality at any cost | **C** with Runway Gen-3 Alpha (~$30 for all 30) |

My honest take: **start with Option A**. It's instant and free. If a
specific archetype looks soft after the crossfade, regenerate just
that one with MiniMax. Don't blanket-regenerate until you've seen Option A.
