## What just shipped (since 2026-05-01)

### Voice mode — full-duplex Conv AI is live

Rwendo's most-bet feature. After multiple sessions fighting `@elevenlabs/react-native` (broken on RN — see [docs/stack/elevenlabs-conversational.md](stack/elevenlabs-conversational.md)) we ripped the official SDK out and built a hand-rolled WebSocket client to ElevenLabs's ElevenAgents endpoint. ElevenLabs support confirmed this is the supported path until their Q1-2026 RN re-architecture lands.

| Piece | File | What it does |
|---|---|---|
| WebSocket client | [lib/conv-ai-ws.ts](../lib/conv-ai-ws.ts) | Pure-JS client. Sends `conversation_initiation_client_data` with overrides (system prompt, first message, language, voice_id), parses every server event (audio, user_transcript, agent_response, agent_response_part, interruption, vad_score, ping). Stack-safe base64 codec — no Buffer dep. |
| Headless hook | [hooks/useConvAISession.ts](../hooks/useConvAISession.ts) | Centralises native audio init + WS lifecycle + memory loading + turn persistence. Consumed by both the full-screen voice route and the inline orb in chat. |
| PCM mic + speaker | `@speechmatics/expo-two-way-audio` | Fixed 16 kHz / 16-bit / mono PCM matches ElevenLabs byte-for-byte. Hardware AEC, noise suppression. New native module → triggered the EAS rebuild. |
| Inline voice UI | [app/(tabs)/companion.tsx](../app/(tabs)/companion.tsx) | Tap voice button → composer is replaced by an inline panel: orb (left, tap to interrupt) → status (middle) → red ✕ hang-up (right). Transcripts feed the same chat list as text. |
| Full-screen route | [app/companion/voice-conv.tsx](../app/companion/voice-conv.tsx) | Legacy entry path retained as a fallback. Same hook. |

**Voice resolution** at session start now follows priority: profile-selected `rwenVoice` → active companion's `defaultVoiceId` → `RWEN_VOICES.male_warm` (George) fallback. Voice-id override is sent in the init message; agent must have **Security → Overrides → Voice** toggled ON in the ElevenLabs dashboard for it to land.

**What was removed**:
- `@elevenlabs/react-native`, `@elevenlabs/elevenlabs-js`, `elevenlabs`, `@livekit/react-native`, `@livekit/react-native-expo-plugin`, `@livekit/react-native-webrtc`
- `components/companion/VoiceImpl.tsx`, `components/companion/voice-polyfills.ts`, `lib/conversational-ai.ts`
- LiveKit expo plugin entry in `app.json`
- well-known-symbols metro shim (no longer needed)

Net result: 50% smaller bundle (build upload was 48.8 MB vs 98.7 MB previously), ~8 min EAS build vs ~25 min, no more polyfill maintenance burden.

### Conversation memory — single-write architecture

Both text chat and voice mode now save through one path: [`appendConversationTurn` in lib/conversation-memory.ts](../lib/conversation-memory.ts). Future avatar/lipsync mode just calls the same helper.

- `loadRecentConversationRecap(userId, n=10)` — formatted dialogue recap injected into the system prompt at session start
- `loadCompanionFacts(userId, k=8)` — top-k from `companion_facts` table, passed as `memoryFacts` slot in the prompt
- `appendConversationTurn(userId, role, content, opts)` — single writer; both text and voice use this; `claude.ts` was refactored to call it instead of inline `supabase.insert`

**Bug fixed**: `loadConversationHistory` was using `order(created_at asc).limit(40)` — returned the FIRST 40 rows forever, hiding everything after the table grew past 40. Switched to `order(created_at desc).limit(40)` then reverse in JS for display.

**Bug fixed**: voice transcripts were silently failing to save because the `session_id` UUID fallback was producing a non-UUID string Postgres rejected. Fallback now returns empty string → undefined → `null` in the insert.

### UI rework — three distinct "clear" actions

Per user spec, three separate destructive paths:

| Where | Label | What it does |
|---|---|---|
| Profile → Account → Delete account | (existing) | Wipes everything — profile, conversations, all data |
| Profile → Account → **Erase saved chat history** | (new) | Wipes only the `conversations` table for this user. Account stays. |
| Chat header → **⌫** | (new) | Clears only on-screen messages. DB untouched. Pulling up the chat next time still has full history. |

Voice bar layout: orb-left, status-middle, hangup-right. Header mic removed; voice still launches via the empty-composer 〰 button.

### Auth + signup — major redesign

New layout for [app/(auth)/sign-up.tsx](../app/(auth)/sign-up.tsx) and [app/(auth)/sign-in.tsx](../app/(auth)/sign-in.tsx):

- **OAuth bar** at top (3 small square buttons in a row OR one full-width button if only one provider is enabled). Wired via [lib/oauth.ts](../lib/oauth.ts). Currently:
  - ✅ Google (active) — uses `expo-auth-session` + Supabase hosted OAuth, no native dep
  - ❌ Apple (hidden behind `OAUTH_READY.apple = false` until Apple Dev account is approved + `expo-apple-authentication` is installed in next EAS build)
  - ❌ Crypto wallet (hidden behind `OAUTH_READY.crypto = false` until WalletConnect SDK is installed; planned to associate with XP token economy on XRPL chain)
- **Email + password form** below the OAuth bar
- **Three consent checkboxes** (down from four):
  1. Combined legal — Terms + Privacy + 13+ + AI disclosure (required, with inline links)
  2. Privacy promise — "Rwendo will not sell or share my data" (required)
  3. Marketing — opt-in for promotional emails (optional, unchecked default — required by GDPR/CASL/PECR/AU Spam Act)

### 6-digit OTP signup verification

Switched from magic-link to 6-digit OTP:
- New screen [app/(auth)/verify-code.tsx](../app/(auth)/verify-code.tsx) — auto-focus, paste/autofill support (iOS one-time-code), auto-submit on 6 digits, resend button
- AuthContext methods `verifySignupOtp` and `resendSignupOtp` added
- Sign-up flow routes to `/verify-code` after submit instead of "check your email and click link"
- Custom branded email template (Rwen waving on blue gradient + 6-digit code in monospace) — image hosted in Supabase Storage `rwendo-email-signup-asset/rwen-waving-400.png` (400×400 PNG, 35 KB, transparent)

### Companions tab — language sync + per-preset translations

- Picking a companion auto-syncs `rwenVoice` to the preset's `defaultVoiceId` (so Maya sounds like Jessica, Tendai like Charlie, etc. for text-mode TTS)
- Companion taglines + descriptions translated across all 5 languages (Shona, French, Chinese, Tagalog, English) — keyed under `companion_presets.<id>.tagline/description` in each speaker pack's `common.json`
- Companions tab UI strings (header, banners, sections, Soul-tier upgrade) translated across all 5 languages

### App language → speaker pack synchronisation

When a user picks a non-English app language in Profile → App Language, **both** `app_language` (i18n) AND `speaker_pack_id` (AI persona language, voice options, greetings, ElevenLabs language code) update simultaneously in DB and SettingsContext. ProfileLoader on app boot now reads `speaker_pack_id` from `profiles` so the pick survives reloads. Earlier the speaker pack reverted to English on every fresh launch — fixed.

### Companion age-gating

`CompanionPreset.ageGate?: number` field added. Aria gets `ageGate: 18`. Three layers of enforcement:

1. **Companions tab filter** — under-18s never see Aria's card. List looks shorter, no "locked" badge, no awareness she exists.
2. **`handleActivate` guard** — even with a tampered client, an alert blocks activation.
3. **Postgres trigger** `enforce_companion_age_gate` — last line of defence; rejects any INSERT/UPDATE setting `companions.preset_id = 'aria'` when the user's DOB shows them under 18.

To add another age-gated preset later: set `ageGate: <num>` in `presets.ts`, add a `WHEN` line to the trigger's CASE statement.

## Database changes since 2026-05-01

### New columns (profiles)

| Column | Type | Why |
|---|---|---|
| `marketing_consent_at` | timestamptz | When user opted in to marketing emails. Required by GDPR/CASL/PECR/Spam Act for any marketing email send. |
| `info_protection_acknowledged_at` | timestamptz | When user ticked the "data stays private" checkbox at signup. Distinct from terms/privacy ToS consent. |

### RPC changes

`record_consents` extended with `p_info_protection` and `p_marketing` params:

```sql
record_consents(
  p_user_id, p_terms, p_privacy, p_age, p_ai_disclosure,
  p_voice, p_info_protection, p_marketing  -- new
)
```

### New RLS policy

`Users can delete own conversations` on `public.conversations` for DELETE (`auth.uid() = user_id`). Lets the "Erase saved chat history" button actually delete; previously RLS silently blocked the DELETE because no policy existed.

### New trigger

`enforce_companion_age_gate` (BEFORE INSERT or UPDATE OF preset_id ON companions). Rejects rows where the preset has an age gate and the user's DOB places them below it.

### Migrations applied

- `add_delete_policy_conversations`
- `add_consent_columns`
- `extend_record_consents`
- `enforce_companion_age_gate`
